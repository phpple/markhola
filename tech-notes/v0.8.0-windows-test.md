# v0.8.0 Windows 11 测试文档

## 一、验证目标

验证 `v0.8.0` 的 Windows 11 版本是否真正覆盖了 `<= v0.7.3` 的**完整累计用户能力集合**，而不是只具备一个可启动、可基础编辑的缩水版本。

本测试文档默认采用更严格的判定标准：  
只要 Windows 缺失任何一个在 `v0.7.3` 及以前已经正式存在的用户能力，就视为当前版本**未达到发布要求**。

## 二、发布门槛

`v0.8.0` 在 Windows 11 上满足以下条件之前，不应视为完成：

1. 应用可以稳定构建并成功启动
2. Windows 提供与 `v0.7.3` 累计范围一致的顶层能力集合
3. 原生菜单、快捷键、文档工作流、渲染工作流、输出工作流在 Windows 上均有可用实现
4. 用户文档不再把 Windows 描述成低配或受限 fallback 路径

## 三、自动验证目标

1. Windows 下 `cargo build` 与 `cargo test` 可以继续通过
2. Windows WebView 壳层修复后，启动路径保持稳定
3. Shell 标记与事件绑定仍然暴露以下入口：
   - open / save / save as
   - documentation / about
   - find panel
   - tab 激活 / 关闭
4. 平台快捷键映射保持正确：
   - macOS 使用 `Command`
   - Windows 使用 `Ctrl`
5. Windows 专用菜单与输出模块在平台条件编译下可以正常通过编译
6. Windows 构建路径不依赖 macOS-only 的 AppKit / WebKit 模块

## 四、手工验证矩阵

### 1. 启动与应用壳层

1. 直接在 Windows 11 上启动应用  
   预期：应用可正常打开，不再出现 WebView2 参数错误或启动崩溃。

2. 从资源管理器双击 Markdown 文件启动应用  
   预期：目标文件直接进入工作区。

3. 从命令行传入 Markdown 路径启动应用  
   预期：目标文件直接进入工作区。

4. 拖拽 Markdown 文件到窗口中  
   预期：文件被成功打开，状态提示正确。

### 2. 原生菜单

1. 检查 Windows 应用是否存在以下顶层菜单：
   - `File`
   - `Edit`
   - `Tab`
   - `Help`

2. File 菜单  
   预期命令包含：Open、Save、Save As、Print、Export PDF、Export HTML、Close、Exit。

3. Edit 菜单  
   预期命令包含：Undo、Redo、Cut、Copy、Paste、Select All、Find。

4. Tab 菜单  
   预期命令包含：Next Tab、Previous Tab、Close Tab、Close Other Tabs、Close All Tabs。

5. Help 菜单  
   预期命令包含：Documentation、About。

6. 菜单禁用状态  
   预期：当没有打开文档时，文档相关命令会正确禁用。

### 3. 快捷键对齐

1. `Ctrl + O`  
   预期：弹出打开文件对话框。

2. `Ctrl + S`  
   预期：保存当前文档。

3. `Ctrl + F`  
   预期：按当前模式打开查找面板。

4. `Ctrl + W`  
   预期：关闭当前标签页；关闭最后一个标签页时，行为与当前正式产品一致。

5. `Ctrl + /`  
   预期：在只读 / 可写模式之间切换。

6. 可写模式编辑器快捷键  
   预期：全选、复制、粘贴、剪切、撤销、重做、行首行尾移动、缩进、反缩进全部可用。

### 4. 文档工作流

1. 打开 `examples\basic.md`  
   预期：内容正常加载。

2. 再打开 `examples\languages.md` 与 `examples\math.md`  
   预期：多个标签页在同一窗口中保留。

3. 在多个标签页之间反复切换  
   预期：激活状态正确切换，每个标签页保持自己的内容和状态。

4. 编辑某个标签页但不保存，再切换出去再切回来  
   预期：dirty 状态和编辑内容都被保留。

5. 执行 Save 与 Save As  
   预期：路径与 dirty 状态更新正确。

### 5. 只读渲染能力

1. 打开 `examples\basic.md`  
   预期：标题、链接、引用、表格、图片等正常渲染。

2. 打开 `examples\languages.md`  
   预期：语法高亮、代码行号、语言徽标正常显示。

3. 打开 `examples\mermaid.md`  
   预期：Mermaid 被渲染为图表，而不是停留在 fenced code。

4. 打开 `examples\math.md`  
   预期：行内公式、`$$...$$`、fenced `math` 块都正常渲染。

### 6. 查找与替换能力

1. 在只读模式下对 `examples\basic.md` 执行查找  
   预期：结果数量稳定，支持 Next / Previous、`Enter`、`Shift + Enter`。

2. 在可写模式下对已编辑文档执行查找 / 替换  
   预期：Replace 与 Replace All 正常工作，并且不会破坏 dirty 状态。

### 7. 输出能力

1. HTML 导出  
   打开 `examples\basic.md`。  
   预期：导出的 HTML 为独立文件，Markdown 样式正常，Mermaid / Math 增强逻辑仍然可用。

2. PDF 导出  
   打开 `examples\pdf-export.md`。  
   预期：Windows 可以成功导出当前激活标签页。  
   预期：标题、表格、图片、代码、Mermaid、数学公式等内容按既有产品能力保留。  
   预期：可写模式下未保存修改会进入导出结果。

3. 打印  
   打开 `examples\basic.md` 与 `examples\mermaid.md`。  
   预期：Windows 打印流程可成功打开，并针对当前激活标签页。  
   预期：打印内容是文档内容而不是应用 shell。  
   预期：可写模式下未保存修改会进入打印结果。

### 8. Documentation 与 About

1. 打开 `Help > Documentation`  
   预期：应用内打开内置发布文档。

2. 打开 `Help > About`  
   预期：About 面板正常显示当前版本与构建信息。

### 9. 主题回归

1. 修改 `themes\default\layout.css`  
   预期：应用可反映主题修改结果。

## 五、阻塞发布的失败条件

出现以下任一情况，都应阻塞 `v0.8.0`：

1. Windows 可以启动，但没有原生菜单
2. PDF 导出或打印仍然回退为 “macOS only”
3. Windows 缺失任何一个已发布快捷键或标签页命令
4. 文档声称支持 Windows，但仍遗漏历史已发布能力
