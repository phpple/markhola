# v0.7.7 Test Document

## Goal

Verify that the footer status bar is a native window component and remains pinned to the bottom of the window.

## Manual Checks

1. Launch MarkHola with no document open.
   - Expected: the footer is pinned to the bottom of the window.
   - Expected: the WebView empty state appears above it.
2. Open a Markdown file.
   - Expected: the footer remains pinned to the bottom.
   - Expected: path, words, lines, mode, and status update correctly.
3. Toggle writable mode with `Command + /`.
   - Expected: the footer remains pinned to the bottom.
   - Expected: mode and status update correctly.
4. Open multiple tabs and switch between them.
   - Expected: footer values update for the active tab only.
5. Switch themes from `View > Theme`.
   - Expected: the native footer styling still feels visually aligned with the active theme.
6. Resize the window taller and shorter.
   - Expected: the WebView content resizes above the footer.
   - Expected: the footer remains pinned to the native window bottom edge.

## Regression Checks

1. Export, print, and HTML/PDF generation still work.
2. Drag-and-drop, open file, and tab switching still work.
3. Empty-state layout does not pull the footer into the WebView area.
