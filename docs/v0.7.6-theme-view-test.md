# v0.7.6 Test Document

## Objective

Verify that `v0.7.6` delivers:

1. four app shell themes
2. a `View` menu with native theme selection
3. fullscreen viewing from the `View` menu

## Automated checks

1. Theme loader resolves the default theme CSS
2. Theme loader resolves each named theme CSS
3. Theme loader falls back safely when a named theme resource is unavailable
4. App shell HTML includes the selected theme CSS
5. Supported theme ids and labels stay stable for menu wiring

## Manual checks

### Theme menu

1. Launch the app
   Expected: the app opens with the `default` theme

2. Open `examples/theme-showcase.md`
   Expected: the document renders normally

3. Use `View > Theme > GitHub`
   Expected: the shell updates immediately to the GitHub-style theme

4. Use `View > Theme > Dark`
   Expected: the shell updates immediately to the dark theme

5. Use `View > Theme > Light`
   Expected: the shell updates immediately to the light theme

6. Return to `View > Theme > Default`
   Expected: the original default shell appearance returns

7. Re-open the `View > Theme` submenu after each change
   Expected: the current theme item is visibly selected

### Readonly and writable behavior

1. Switch themes in readonly mode
   Expected: document content remains visible and the theme updates

2. Toggle to writable mode
   Expected: the editor remains usable

3. Switch themes again in writable mode
   Expected: the editor chrome and surrounding shell update without losing content

### Fullscreen

1. Use `View > Toggle Full Screen`
   Expected: the app enters fullscreen

2. Use `View > Toggle Full Screen` again
   Expected: the app exits fullscreen

3. Enter fullscreen with an opened document
   Expected: the active document remains open and usable

4. Enter fullscreen after switching to a non-default theme
   Expected: the selected theme remains active

5. Toggle to writable mode while fullscreen is active
   Expected: editing still works

## Release sandbox verification

Before publishing the GitHub release:

1. build the DMG candidate
2. mount the DMG in the sandbox
3. launch the packaged app copy
4. open `examples/theme-showcase.md`
5. verify all four themes from `View > Theme`
6. verify fullscreen toggling from `View`
7. verify theme switching still works while a document is open
