# v0.7.7 Technical Design

## Scope

Fix the bug where dragging multiple Markdown files into MarkHola does not open them as separate tabs.

## Expected Behavior

1. Dragging one Markdown file into the app keeps the current behavior.
2. Dragging multiple Markdown files into the app opens every dropped file.
3. Each opened file appears in its own tab.
4. If one dropped file is already open, MarkHola should activate the existing tab instead of duplicating it.

## Implementation Notes

1. Keep the existing single-document workspace behavior unchanged.
2. Update the macOS opened-files event handling so it dispatches every dropped file path instead of only the first one.
3. Ignore non-file URLs in the opened-files event.
4. Reuse the existing `OpenPath` flow for each file so shell readiness, asset registration, and tab activation continue to work consistently.

## Non-Goals

- Changing the tab order rules beyond the current append-and-activate behavior
- Adding new drag-drop UI affordances
